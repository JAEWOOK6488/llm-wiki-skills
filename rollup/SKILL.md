# Rollup — 세션 작업을 raw 소스로 증류 (crystallization)

탐색·디버깅·작업 같은 **완결된 작업 흐름**도 아티클·논문과 똑같은 소스다. 이 스킬은 세션을 증류해 `raw/`에 회고 노트로 스테이징한다. 그 다음 `/ingest`가 위키로 편입한다.

> [!important] 절대 원칙
> - **기존 raw 파일 수정 금지.** rollup은 **새 raw 파일만 생성**한다. 사용자가 쓴 데일리노트 등 기존 파일을 덮어쓰거나 append하지 않는다(raw 불변, CLAUDE.md §3).
> - **민감정보 필터 (privacy)**: API 키·토큰·비밀번호·PII·비공개 경로·자격증명은 증류에서 **제거**한다. 위키에 닿기 전에.
> - **ingest는 하지 않는다.** raw에 스테이징만 하고 `/ingest`를 안내한다(tag→ingest처럼 분리). 사용자가 "ingest까지"라고 하면 그때 ingest.

핵심 기조: **묻지 말고 보여주기** — 증류해서 파일로 보여주고, 사용자는 Obsidian/ingest 단계에서 교정.

## 무엇을 어떻게 증류하나

### 현재 세션 (기본)
- 에이전트는 **지금 이 대화 컨텍스트에서 직접 증류**한다 — 트랜스크립트 파일을 읽을 필요 없음(자기 세션이므로).
- 맥락 보강(선택): 이 세션 동안의 git 변경을 근거로. 예: `git -C <repo> log --since='<세션 시작 추정>' --pretty=oneline`, 필요시 `git diff --stat`.
- **"한 일"이 아니라 "건진 것"** 중심으로 증류. todo·잡담·시행착오 과정은 제외, 재사용 가치만.

증류 카테고리:
- 🧭 **한 일(활동 뼈대)** — 맥락용 한두 줄 (git 커밋 등)
- 🧠 **건진 지식** — 재사용 가능한 사실 → ingest 시 [[엔티티]]/[[개념]]/[[thread]]로 승격
- 🔧 **교훈 · 함정** — 막혔다 푼 것 + 원인/해법
- ⚙️ **워크플로우 · 도구 인사이트** — "이거 쓰니 좋더라". 재참조 팁 → wiki / 반복 선호 → **memory 승격 후보 ⭐**
- ❓ **미해결** — 다음에 조사할 것
- ✅ **후속/할 일**

### 오늘 전체 (옵션: "오늘 전체" / `--day`)
- 오늘 수정된 모든 세션 트랜스크립트(`~/.claude/projects/**/*.jsonl`, `-newermt 'today'`) + 작업 레포들의 오늘 git 커밋을 모아 하루 단위로 증류.
- 여러 세션의 중복은 합치고, 시간 흐름을 살린다.

## 대상 vault · 파일 위치
- **기본 대상**: `obs_daily`(작업-일 기록). 사용자가 `obs_work` 등 다른 vault를 지정하면 그쪽.
  - 도메인 지식 중심 세션이면 `obs_work/raw/`가 맞을 수 있음 — 모호하면 한 번만 확인.
- **파일**: `raw/daily/<YYYY>/<MM>/<YYYY-MM-DD>.md`
  - 그 파일이 **없으면** 생성(데일리노트 형식 채워서).
  - **있으면 절대 덮어쓰지 말고** 같은 폴더에 `<YYYY-MM-DD>-session-<HHMM>.md` 로 **별도 파일** 생성.
- **frontmatter** (raw 스키마 준수):
  ```yaml
  title: "<YYYY-MM-DD> 세션 롤업"
  source: "claude-code session (rollup)"
  created: <오늘>
  tags: [worknote, rollup]
  generated: rollup       # 자동 생성 표시
  ingested: false
  ```

## 절차
1. 범위 결정(현재 세션 / 오늘 전체). 현재 세션이면 대화 컨텍스트에서 증류, 오늘 전체면 트랜스크립트+git 수집.
2. **민감정보 필터** 적용.
3. 대상 vault·파일 결정(위 규칙 — 기존 파일 보호).
4. 증류 노트를 **새 raw 파일로 생성**.
5. 보고 + `/ingest` 안내. (자동 ingest 안 함)

## 완료 보고 형식
- 📝 만든 raw 파일 경로
- 🔎 증류 요약(섹션별 핵심 미리보기)
- ⚙️ memory 승격 후보(반복 선호로 보이면) — "저장할까요?" 제안
- 🚫 필터링한 민감정보가 있었으면 "민감정보 N건 제외" 한 줄
- 🚀 다음 단계: `/ingest`로 위키 편입 안내

사용자는 raw 파일을 보고 교정한 뒤 `/ingest` 한다.
