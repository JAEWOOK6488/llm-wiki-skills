# Lint — 볼트 건강검진

`wiki/`(승격된 노트)와 `moc/`·`index.md`·`overview.md` 등 LLM이 소유한 산출물의 일관성을 점검·정비한다.
이 볼트의 `CLAUDE.md` §6 Lint를 실행 절차로 구체화한 것. 충돌 시 `CLAUDE.md`를 따른다.

> [!important] 절대 원칙
> - **`raw/`는 절대 수정 금지** (원본 불변). raw는 **읽기만** 한다. raw에 깨진 링크/누락이 보여도 고치지 말고 보고만.
> - **모든 자동 수정은 리포트의 `✅ 수정됨(자동)` 섹션에 기록**한다 — 파일 / 무엇을 / before→after. 추적·되돌리기 가능해야 함.
> - **삭제는 하지 않는다.** stale도 사용자 결정 전까지 보존. orphan은 연결 제안만.

핵심 기조: **묻지 말고 보여주기** — 단, 안전한 것만 자동 수정한다. 모호하거나 파괴적인 변경(삭제·stale 처리)은 사용자에게 묻는다.

## 검진 대상
- 대상: `wiki/**`, `moc/**`, `index.md`, `overview.md`, `_tags.md` (LLM 소유 영역).
- 제외(읽기 전용 참조): `raw/**`(불변), `CLAUDE.md`·`_templates/**`(스키마/템플릿), `log.md`.

## 절차

### 0. 전수 스캔 (읽기)
- 대상 영역의 모든 `.md`를 모아 **페이지 목록**(파일명=표제어)과 **모든 `[[위키링크]]`**(출처 파일 → 링크 대상)를 수집한다.
- 링크 해석(Obsidian 규칙): `[[Name]]`, `[[Name|별칭]]`, `[[Name#헤딩]]` → 모두 `Name.md`로 해석. 경로 무관(파일명만 일치하면 연결).

### 1. Broken link — 자동 재연결
- **깨진 링크** = 대상 페이지가 실제로 존재하지 않는 `[[링크]]`. (단, 아래 "미생성"은 제외)
- **재연결 시도**:
  1. **파일명 매칭**: 대소문자·공백·하이픈 차이, 부분일치, 알려진 동의어(`ros2`↔`ROS 2` 등 `_tags.md`/기존 표제어 참고).
  2. **내용 유사도**: 링크 문맥이 가리키는 개념과 가장 가까운 기존 페이지.
- **고신뢰 단일 후보**가 있으면 → **링크를 올바른 표제어로 자동 교체**하고 `✅ 수정됨`에 기록(`[[잘못된]]` → `[[올바른]]`).
- 후보가 **여러 개로 모호**하거나 **없으면** → 자동수정하지 않고 `⚠️ 수동 확인 필요`에 기록 + 사용자에게 알림(추천 후보가 있으면 함께).
- **미생성(만들 거리)** 은 broken이 아니다: 빈 링크 `[[ ]]`, MOC의 `(예정) [[X]]`처럼 **의도적 미생성 표시**는 `📝 미생성` 목록으로 따로 보고(자동 생성하지 않음 — 페이지 생성은 ingest/query의 몫).
- 링크가 `raw/` 파일 안에 있으면 **고치지 말고 보고만**.

### 2. Frontmatter 누락/불완전 — 자동 수정
- 모든 wiki 페이지는 frontmatter를 갖춰야 한다(`CLAUDE.md §4`). 누락·불완전 필드를 **자동으로 채운다**:
  - `type`: 폴더로 추론(`sources/`→source, `entities/`→entity, `concepts/`→concept, `moc/`→moc, `synthesis/`→synthesis).
  - `title`: H1 또는 파일명.
  - `created`/`updated`: 없으면 보수적으로 채움(파일 mtime 또는 오늘). 기존 값은 보존.
  - `tags`: **위키에서 유사 태그를 찾아 보강** — `_tags.md` 레지스트리 + 같은 주제 이웃 페이지(링크로 연결된 페이지)의 태그 + 본문 키워드를 대조해 빠진 주제 태그를 채운다. 동의어 신설 금지(`CLAUDE.md §4` 태그 규약).
  - `status`(stub/draft/mature), `sources`(근거 소스 수) 등 누락 시 합리적 기본값.
- 채운 내용을 `✅ 수정됨`에 기록(어느 필드를 무엇으로).
- **`raw/` frontmatter 누락은 자동수정 대상 아님**(raw 불변). 보고만 하고 "tag/ingest 스킬로 처리" 안내.

### 3. Stale — 삭제 여부 사용자 결정
- **stale** = 최신 소스가 뒤집은 옛 주장, 더 이상 유효하지 않은 내용, 또는 해소되지 않은 모순. (log/source 날짜·모순 콜아웃·의미 분석으로 판단)
- **자동 삭제·수정 금지.** 발견 항목을 `❓ 결정 필요(stale)`에 모아 사용자에게 **지울지 / 갱신할지 / 유지할지**를 묻는다(AskUserQuestion 등). 결정 후에만 반영.

### 4. Orphan — 연결 제안 (삭제 금지)
- **orphan** = inbound 링크가 하나도 없는 wiki 페이지(index/MOC/다른 페이지 어디서도 안 가리킴).
- **자동 연결 제안**: 어느 MOC/페이지에 어떤 맥락으로 링크하면 좋을지 구체적으로 제안(`🔗 연결 제안`). index.md 등록 누락처럼 명백한 건 제안과 함께 자동 반영해도 되나, 그 경우도 `✅ 수정됨`에 기록.
- **삭제하지 않는다.** 연결은 제안하고, 적용은 사용자 합의(또는 명백한 경우 자동+기록).

### 5. 리포트 + 로그
리포트 섹션(아래 "완료 보고 형식")을 사용자에게 보여주고, `log.md` 맨 위에 엔트리 append:
```
## [YYYY-MM-DD] lint | <요약: broken N / frontmatter N / stale N / orphan N>
- 수정됨(자동): ...
- 결정 필요/제안: ...
```

## 완료 보고 형식
- 🩺 **요약**: broken link N(자동수정 M / 수동 K) · frontmatter 보강 N · stale N · orphan N · 미생성 N
- ✅ **수정됨(자동)** — 추적용. 파일별 `무엇: before → after`
  - broken link 재연결 / frontmatter 필드 채움 / (명백한) orphan 연결
- ❓ **결정 필요 — stale**: 항목별로 지울지/갱신할지/유지할지 질문
- 🔗 **연결 제안 — orphan**: "이 페이지를 [[MOC]]에 ~ 맥락으로 링크" 제안
- ⚠️ **수동 확인 — broken link**: 매칭 후보 없음/모호. (추천 후보 있으면 표기)
- 📝 **미생성(만들 거리)**: 의도적 빈 링크·`(예정)` placeholder 목록
- 🚫 raw 관련 발견은 "raw 불변 — 수정 안 함"으로 보고만.

사용자는 Obsidian에서 결과를 보며 교정하고, stale 결정·orphan 연결을 확정한다.
