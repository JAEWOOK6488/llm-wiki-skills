# Ingest — 소스를 LLM Wiki로 정리

이 스킬은 이 볼트의 `CLAUDE.md` §6 Ingest 워크플로우를 실행 가능한 절차로 구체화한 것이다.
규약의 단일 진실 원천은 `CLAUDE.md`다. 충돌 시 `CLAUDE.md`를 따른다.

핵심 기조: **묻지 말고 보여주기.** 진짜로 막히지 않는 한 되묻지 않고 먼저 위키에 반영한 뒤 결과를 보여준다.

## 대상 선정

- 사용자가 특정 파일을 지목하면 그것을 ingest.
- "새로 넣은 거 정리해줘"처럼 지목이 없으면 `raw/articles/`(및 `raw/`)에서 아직 `wiki/sources/`에 요약이 없는 파일을 찾아 대상으로 삼는다. 여러 개면 한 건씩 순차 처리하고, 목록을 먼저 보여준다.
- URL을 ingest하라고 하면, 외부 소스는 변하므로 먼저 본문을 `raw/articles/`에 마크다운 스냅샷으로 저장한 뒤 그 파일을 ingest한다.

## 절차

### 1. 읽기
- 소스를 읽는다. 이미지가 섞인 마크다운이면 텍스트를 먼저 읽고, 이해에 필요한 이미지만 별도로 view 한다.

### 2. 소스 요약 페이지 작성 — `wiki/sources/`
- `_templates/source.md` 골격으로 `wiki/sources/<제목>.md` 생성.
- frontmatter: `type: source`, `created`/`updated`(오늘), `tags`, `status`, `source_path: raw/articles/<원본>`.
- 본문: 한 줄 요지 → 요약 → 주요 인용/데이터 → 등장 엔티티·개념(`[[링크]]`) → 기존 위키와의 연결/모순.

### 3. 엔티티·개념 페이지 생성/갱신 — `wiki/entities/`, `wiki/concepts/`
- 소스에 등장한 고유 대상(제품·도구·시스템·사람)은 entity, 일반 원리·기법·이론은 concept.
- **중복 방지**: 새로 만들기 전에 `index.md`를 확인. 이미 있으면 새로 만들지 말고 **기존 페이지를 갱신**(새 사실 추가, `updated` 갱신, `sources` +1, 근거 소스에 `[[이번 소스]]` 추가).
- 한 소스가 보통 5~15개 페이지를 건드린다. 빠뜨리지 말 것.
- 본문에서 언급한 다른 엔티티·개념은 페이지가 아직 없어도 `[[빈 링크]]`로 연결해 둔다(만들 거리 표시).

### 4. 모순 처리
- 새 정보가 기존 주장과 충돌하면 **삭제하지 말고** 양쪽 출처를 함께 적고 `> [!warning] 모순` 콜아웃으로 명시.

### 5. MOC 연결
- 이번에 만든/갱신한 페이지가 속하는 주제 MOC(`moc/VLA.md`, `moc/임베디드 모터제어.md`, `moc/전자공학 회로설계.md`, `moc/로봇공학 동역학 기구학.md`)에 링크를 추가. `(예정)` 자리표시자가 있으면 실제 링크로 교체.
- 적합한 MOC가 없고 새 주제가 3~5개 페이지로 커졌으면 `_templates/moc.md`로 새 주제 MOC를 만들고 `moc/home.md`에 등록.

### 6. index 갱신 — `index.md`
- 새로 만든 모든 페이지를 해당 카테고리(Sources/Entities/Concepts)에 `- [[페이지]] — 한 줄 요약`으로 추가.
- `index.md` frontmatter의 `updated`도 오늘로.

### 7. overview는 조건부 갱신 — `wiki/overview.md`
- **매번 갱신하지 않는다.** 이번 소스가 전역 thesis·주요 주제·미해결 질문·모순을 실제로 바꿀 때만 해당 섹션을 갱신.
- 바꿨으면 무엇을 왜 바꿨는지 한 줄로 사용자에게 보고.

### 8. 로그 — `log.md`
- 맨 위(최신이 위)에 엔트리 append:
  ```
  ## [YYYY-MM-DD] ingest | <소스 제목>
  - 생성: [[...]], [[...]]   갱신: [[...]]
  - 연결 MOC: [[...]]   overview 갱신: 예/아니오
  ```

## synthesis는 ingest 범위 밖
- `wiki/synthesis/`(교차주제 종합)는 ingest가 자동 생성하지 않는다. 그건 Query 워크플로우에서 사용자 질문에 답하며 환류하는 산출물이다. ingest 중 좋은 synthesis 후보가 보이면 만들지 말고 마지막 보고에 "후보"로만 제안한다.

## 완료 보고 형식
ingest를 마치면 다음을 간결히 보여준다:
- 📄 만든 소스 요약: `[[...]]`
- 🆕 새 페이지: entities `[[...]]`, concepts `[[...]]`
- ♻️ 갱신 페이지: `[[...]]`
- 🗺️ 연결한 MOC / ⭐ overview 변경 여부
- 💡 (있으면) synthesis 후보·다음에 조사하면 좋을 질문

사용자는 Obsidian에서 결과를 보며 교정한다.
